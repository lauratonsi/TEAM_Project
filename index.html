How does it work? — RAG architecture

The Virtual Analyst is a RAG (Retrieval-Augmented Generation) system built on 320 text chunks extracted from the 30 validated XML files. Each query uses hybrid BM25 + vector search (FAISS + all-MiniLM-L6-v2) with Reciprocal Rank Fusion, detects intent (hotels / transport / attractions / safety) and responds in real time without an external LLM.

⚠️ It knows only the 30 capitals in the dataset; some lack hotel data because they are absent from the Wikivoyage sources.
